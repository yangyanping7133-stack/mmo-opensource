## Context

### Project Background (unchanged)
- **Model**: Qwen3-32B, inference framework vLLM-Ascend (Ascend NPU)
- **Hardware**: 8 x Ascend NPU
- **Core Objective**: CoT token reduction >= 30%, latency reduction >= 20%
- **Hard Constraint**: Accuracy must not fall below baseline
- **Constraints**: Training-free preferred (only threshold/prompt tuning allowed); extremely low-cost SFT acceptable (<=1K samples)
- **Evaluation Scale**: 5 datasets, 1270 questions, 128 concurrency

### ADR-001 Review and Lessons Learned

| Method | Result | Lesson |
|--------|--------|---------|
| TALE | Pilot accuracy collapsed (GPQA 60% vs 94%) | Fixed token budget not suitable for datasets with large difficulty variance |
| Concise Prompt | GSM8K 10-sample 90% (maintained) | Simple prompt effective but not systematic enough |
| max_tokens truncation | Accuracy degraded | Coarse-grained truncation not viable |
| DEER | Implemented as comparison baseline | Confidence-based exit unreliable |

### 2026 Paper Insight Summary

22 papers from 2026, key findings:

1. **CRISP** (ACL 2026): Uses `</think` attention anchors, 50-60% compression, no accuracy drop
2. **ASC**: Steering vector injection, 67% compression, 32B verified, 100 samples
3. **SAT** (ACL 2026 Main): FSM 4-mode step-level pruning, <=40% compression
4. **Whisper**: Black-box persuasive prompt, ~40% compression, Qwen3 series verified
5. **DTR**: Not a compression method, but a reasoning quality metric, usable as evaluation indicator

## Decision: Three-Track Parallel + Cascading Fallback

### Track A: ASC — Steering Vector Injection (Primary)
- **Paper**: Activation-Steered Compression (arXiv 2507.04742)
- **Principle**: 100 verbose/concise sample pairs -> extract steering vector -> inject into hidden layers at inference time
- **Expected**: 50-67% token reduction, no accuracy drop, 2.7x end-to-end speedup
- **Advantages**:
  - 32B model already verified
  - MATH500 + GSM8K verified (aligns with our eval suite)
  - Only 100 sample pairs needed, zero training
  - Code open-sourced (https://github.com/ArminAzizi98/ASC)
- **Risks**:
  - Requires modifying inference pipeline to inject steering vector
  - Unknown whether vLLM-Ascend supports hidden state manipulation
  - **Mitigation**: vLLM-Ascend compatibility PoC first (Day 1)
- **Go/No-Go**: Day 1 complete PoC, confirm steering vector can be injected -> Go

### Track B: CRISP — Intrinsic Saliency Pruning (Parallel Primary)
- **Paper**: CRISP (arXiv 2604.17297, ACL 2026 Findings)
- **Principle**: Use `</think` termination token attention patterns -> identify redundancy -> atom-level compression
- **Expected**: 50-60% token reduction, no accuracy drop
- **Advantages**:
  - ACL 2026 peer-reviewed
  - Code open-sourced
  - Cross-model validation
  - Atom-level operations preserve logical coherence (not coarse truncation)
- **Risks**:
  - Requires access to attention weights
  - Unknown whether vLLM-Ascend exposes attention weights
  - **Mitigation**: vLLM-Ascend attention access PoC first (Day 1)
- **Go/No-Go**: Day 1 complete PoC, confirm attention weights accessible -> Go

### Track C: Whisper — Black-box Persuasive Prompting (Fallback + Day 1 Quick Validation)
- **Paper**: Whisper (arXiv 2510.10528)
- **Principle**: Iteratively optimize "persuasive prompts" to guide model toward more concise reasoning
- **Expected**: ~40% token reduction, no accuracy drop
- **Advantages**:
  - Pure prompt method, zero invasiveness, 100% vLLM-Ascend compatible
  - Qwen3 series verified
  - Can start immediately, results on Day 1
  - Can be stacked with any other method
- **Risks**:
  - Effectiveness may vary across datasets
  - Compression ratio may be insufficient (target >= 30%, Whisper may be borderline)
- **Go/No-Go**: Day 1: 50-question quick validation, token reduction >= 25% and accuracy >= baseline -> Go

### Fallback Plan

| Fallback Level | Method | Trigger Condition |
|----------------|--------|-------------------|
| F1 | SAT (step-level FSM pruning) | Both Track A+B are No-Go |
| F2 | Hint Tuning (1K SFT) | All Tracks A+B+C underperform |
| F3 | Dynamic Early Exit | All methods underperform, maximize compression while preserving accuracy |

## Consequences

### Positive
- Based on latest 2026 research, methods more mature
- Three-track parallel reduces risk: at minimum Whisper pure prompt can produce results immediately
- DTR metric provides scientific evaluation framework
- ASC and CRISP have open-source code, low implementation cost

### Negative
- ASC/CRISP both depend on specific vLLM-Ascend capabilities, PoC risk exists
- Running 3 tracks simultaneously requires more parallel resources
- 2026 papers are generally new, reproduction may encounter issues

### Key Changes from ADR-001
1. TALE -> Proven infeasible, fully removed
2. CRISP (2026) -> Newly discovered, fine-grained pruning based on attention anchors
3. ASC (2026) -> Newly discovered, steering vector approach, 32B verified
4. Whisper (2026 update) -> Replaces original concise prompt, more systematic
5. DTR -> Added as evaluation metric (not a compression method)
6. Concurrency 256 -> 128 (hardware stability consideration)
